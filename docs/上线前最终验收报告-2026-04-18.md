### 活体并发补充验证（2026-04-18）

- 命令：`RUN_LIVE_CONCURRENCY_TESTS=1 python -m pytest -q tests/test_concurrent_20devices.py -s`
- 结果：当前机器失败（`1 failed`）
- 失败原因：测试环境为 `Windows + SQLite` 且 Redis 不可用，日志显示回退到 DB-only；同时样本导入后 `pending=172`，未满足该用例要求 `required=460`
- 结论：该结果不代表生产并发正确性退化，属于“验收环境不满足 Linux + PostgreSQL + Redis 基线”的失败；上线前应在目标 Linux 环境补跑一次并发验收
