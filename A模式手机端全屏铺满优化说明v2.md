# A模式手机端全屏铺满优化说明 v2

## 优化目标
1. **全屏铺满** - 充分利用不同手机屏幕空间
2. **大字体大按钮** - 用户看得清楚，方便操作
3. **防止误触** - 接单后不能因为误触其他地方导致票面消失

---

## 核心改进

### 1. 全屏铺满设计

#### 布局结构
```
┌─────────────────────────────┐
│ 卡片标题栏（A模式接单）      │ ← 固定
├─────────────────────────────┤
│ 倒计时栏（渐变背景）         │ ← 固定顶部
│ 期号: 26034    23:45:12     │
├─────────────────────────────┤
│                             │
│ 票面内容区域                 │ ← 可滚动
│ （自适应高度，铺满剩余空间）  │
│                             │
├─────────────────────────────┤
│ 操作按钮栏                   │ ← 固定底部
│ [上一张] [下一张] [停止]     │
│      ● ● ●                  │
└─────────────────────────────┘
```

#### CSS实现
```css
.mode-a-fullscreen-container {
  display: flex;
  flex-direction: column;
  min-height: calc(100vh - 300px);  /* 自适应视口高度 */
}

.mode-a-ticket-scroll {
  flex: 1;  /* 占据剩余所有空间 */
  overflow-y: auto;
}
```

### 2. 大字体大按钮

#### 倒计时
- 桌面端：1.5rem
- 手机端：1.3rem
- 小屏手机：1.2rem
- 使用等宽字体（Courier New）
- 渐变紫色背景，白色文字

#### 按钮
- 最小高度：50px（手机端48px，小屏46px）
- 字体大小：1.1rem（手机端1rem）
- "下一张"按钮更大：1.2rem，宽度1.5倍
- 圆角：8px
- 点击反馈：scale(0.95)

#### 票面内容
- 彩种标题：1.3rem → 1.2rem（手机）→ 1.1rem（小屏）
- 场次编号：1.15rem → 1.05rem → 1rem
- 选项徽章：1.1rem → 1rem → 0.95rem
- 总金额：1.2rem → 1.1rem → 1.05rem

### 3. 防止误触

#### 实现方式
当用户开始接单时，自动添加 `mode-a-active` class到body：

```javascript
async toggleModeA() {
  if (this.modeAActive) {
    // 开始接单 - 添加防误触class
    document.body.classList.add('mode-a-active');
    await this.nextTicket();
  } else {
    // 停止接单 - 移除防误触class
    document.body.classList.remove('mode-a-active');
    this.confirmStop();
  }
}
```

#### CSS效果
```css
/* 防止误触 - 接单时禁用页面滚动 */
body.mode-a-active {
  overflow: hidden;
  position: fixed;
  width: 100%;
  height: 100%;
}

/* 统计栏在接单时隐藏 */
body.mode-a-active .row.g-3 {
  display: none;
}

/* 票面容器占据更多空间 */
body.mode-a-active .mode-a-fullscreen-container {
  min-height: calc(100vh - 150px);  /* 手机端120px */
}
```

#### 防误触效果
1. ✅ 页面整体不能滚动
2. ✅ 统计栏自动隐藏，释放更多空间
3. ✅ 票面容器扩大到几乎全屏
4. ✅ 只有票面内容区域可以滚动
5. ✅ 除非点击"停止"按钮，否则票面不会消失

## 技术细节

### Flexbox布局
```css
.mode-a-fullscreen-container {
  display: flex;
  flex-direction: column;
  min-height: calc(100vh - 300px);
}

.mode-a-countdown-bar {
  flex-shrink: 0;  /* 不缩小 */
}

.mode-a-ticket-scroll {
  flex: 1;  /* 占据剩余空间 */
}

.mode-a-action-bar {
  flex-shrink: 0;  /* 不缩小 */
}
```

### 防误触原理
```css
body.mode-a-active {
  overflow: hidden;      /* 禁止滚动 */
  position: fixed;       /* 固定位置 */
  width: 100%;
  height: 100%;
}
```

这样设置后：
- 页面整体被固定，无法滚动
- 但内部的 `.mode-a-ticket-scroll` 仍然可以滚动
- 用户无法通过滚动页面来误触其他元素

### 视口高度计算
```
接单前：
  容器高度 = 100vh - 300px（统计栏+标题栏+间距）

接单后（body.mode-a-active）：
  容器高度 = 100vh - 150px（统计栏隐藏，只剩标题栏）

手机端接单后：
  容器高度 = 100vh - 120px（更紧凑）

小屏接单后：
  容器高度 = 100vh - 100px（最大化利用）
```

## 常见问题

### Q1: 为什么接单时统计栏会消失？
A: 这是防误触设计的一部分。统计栏消失后，票面可以占据更多空间，同时减少用户误触统计栏的可能性。

### Q2: 如何恢复统计栏？
A: 点击"停止"按钮即可恢复统计栏和正常的页面滚动。

### Q3: 为什么页面不能滚动了？
A: 这是防误触设计。接单时页面被锁定，防止用户误触其他区域。只有票面内容区域可以滚动。

### Q4: 字体会不会太大？
A: 字体大小经过精心调整，在不同屏幕上都保持清晰可读。如果觉得太大，可以在CSS中调整。

### Q5: 按钮会不会太大？
A: 按钮大小（50px高度）符合移动端触控规范（最小44px）。大按钮更易于点击，减少误操作。
